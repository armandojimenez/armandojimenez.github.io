## ğŸ” Research Findings

### Key Requirements Discovered

**Data Collection:**
- Must disclose ALL data collection, even if minimal
- Include both user-provided and automatically collected data
- Specify exact third-party services and their data access

**Advertising:**
- Google AdMob requires specific disclosures
- Must explain Advertising ID usage (IDFA/GAID)
- Must provide opt-out instructions
- Cannot use personalized ads for children

**In-App Purchases:**
- One-time purchases must be clearly described
- Refund policy must be stated (even if "no refunds")
- Must link to Apple/Google refund processes
- Cannot be misleading about what's purchased

**Offline Apps:**
- Must still disclose if third-party SDKs collect data
- Local storage should be explained
- Device backups should be mentioned
- Must clarify that no sync occurs

**Disclaimers:**
- Calculator apps need accuracy disclaimers
- Must state not professional advice
- Should include liability limitations
- Need "as is" disclaimers

## ğŸ“ What Was Learned

### App Store Compliance
1. **Privacy Nutrition Labels** (Apple) must match actual data collection exactly
2. **Data Safety Section** (Google) requires honest disclosure of all third-party SDKs
3. Both stores prioritize **user transparency** over legalese
4. **Contact information** must be easily accessible and responsive

### Legal Requirements
1. **GDPR** requires explicit consent mechanisms and clear withdrawal options
2. **CCPA** focuses on disclosure and user control, not just consent
3. **COPPA** has strict rules for apps accessible by children, even if not targeted at them
4. **Disclaimers** are critical for calculator/tool apps to limit liability

### Design Insights
1. **Mobile-first** is essential - most users will read on phones
2. **Dark mode** is expected, not optional
3. **Table of contents** significantly improves navigation
4. **Performance** matters - policies should load in < 2 seconds
5. **Typography** affects readability more than color choices

### Third-Party Services
1. **Google AdMob** collects more data than developers realize
2. Must link to third-party privacy policies
3. Can't just say "we use ads" - must explain what data is shared
4. Premium/paid versions eliminate most tracking concerns
